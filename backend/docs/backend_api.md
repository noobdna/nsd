# NSD – Backend API Design

## Overview
This document describes the backend API design for NSD (Network Suspicious Detection).

The backend API is responsible for receiving device data, managing alerts and cases,
handling users and authentication, and providing data to the frontend dashboard.

---

## Main Responsibilities

The Backend API handles:

- Device registration
- Data ingestion from IoT / GPS / cameras / sensors
- Event storage and retrieval
- Risk scoring requests
- Alert generation and management
- Case management
- User and role management
- Dashboard data delivery
- Audit logging

---

## API Style

Recommended API style:

- REST API for standard business operations
- JSON request / response format
- HTTPS only
- Token-based authentication
- Role-based access control (RBAC)

Optional future extension:

- WebSocket for real-time alerts
- GraphQL for flexible dashboard queries

---

## Main API Modules

### 1. Authentication API
Handles login, token validation, and user session management.

Example endpoints:

- `POST /api/v1/auth/login`
- `POST /api/v1/auth/logout`
- `GET /api/v1/auth/me`

Main functions:

- User authentication
- Session/token validation
- Role retrieval

---

### 2. Device API
Handles device registration and management.

Example endpoints:

- `POST /api/v1/devices`
- `GET /api/v1/devices`
- `GET /api/v1/devices/{device_id}`
- `PATCH /api/v1/devices/{device_id}`

Main functions:

- Register new devices
- Update device metadata
- Check device status
- Link devices to users or cases

---

### 3. Event Ingestion API
Receives data from IoT, GPS, and other external systems.

Example endpoints:

- `POST /api/v1/events`
- `POST /api/v1/events/bulk`

Main functions:

- Receive telemetry and event data
- Validate payloads
- Store raw and normalized event data
- Trigger detection pipeline

---

### 4. Alert API
Handles suspicious alerts generated by detection logic.

Example endpoints:

- `GET /api/v1/alerts`
- `GET /api/v1/alerts/{alert_id}`
- `PATCH /api/v1/alerts/{alert_id}`
- `POST /api/v1/alerts/{alert_id}/acknowledge`

Main functions:

- Retrieve alerts
- Update alert status
- Assign severity
- Acknowledge or escalate alerts

---

### 5. Case Management API
Handles investigations and case tracking.

Example endpoints:

- `POST /api/v1/cases`
- `GET /api/v1/cases`
- `GET /api/v1/cases/{case_id}`
- `PATCH /api/v1/cases/{case_id}`

Main functions:

- Create investigation cases
- Link alerts to cases
- Assign operators
- Add notes and status updates

---

### 6. Dashboard API
Provides aggregated data for frontend monitoring dashboards.

Example endpoints:

- `GET /api/v1/dashboard/summary`
- `GET /api/v1/dashboard/map`
- `GET /api/v1/dashboard/metrics`

Main functions:

- Summary statistics
- Alert counts by severity
- Active device status
- Map and timeline data

---

### 7. User / Role API
Handles user accounts and permissions.

Example endpoints:

- `GET /api/v1/users`
- `GET /api/v1/users/{user_id}`
- `POST /api/v1/users`
- `PATCH /api/v1/users/{user_id}`

Main functions:

- User management
- Role assignment
- Organization/team mapping
- Permission checks

---

### 8. Audit Log API
Stores and retrieves important operation logs.

Example endpoints:

- `GET /api/v1/audit-logs`
- `GET /api/v1/audit-logs/{log_id}`

Main functions:

- Record operator actions
- Record admin changes
- Support compliance and traceability

---

## Example Request Flow

1. Device sends GPS/event data to Event Ingestion API
2. Backend validates and stores the data
3. Detection engine evaluates suspicious behavior
4. Alert is generated if risk threshold is exceeded
5. Alert appears in dashboard
6. Operator creates or updates a case
7. All actions are stored in audit logs

---

## Security Considerations

- All APIs must require HTTPS
- Authentication should use secure tokens
- Access should be restricted by role
- Sensitive data should be encrypted at rest and in transit
- API requests should be logged for audit purposes
- Rate limiting should be applied to ingestion endpoints

## Summary
The Backend API is the operational core of NSD.
It connects devices, detection logic, alerts, dashboard views, and case workflows
into one manageable platform.
