NSD – Dashboard Documentation

Overview

The NSD Dashboard is the main user interface for monitoring events, alerts, investigations, and system status in the NSD (Network / Neighborhood Suspicious Detection) platform.

The dashboard provides operators, investigators, and administrators with real-time visibility into system activity, suspicious behavior, alerts, and ongoing cases.

The dashboard is designed to be:
	•	real-time oriented
	•	investigation friendly
	•	operationally clear
	•	multi-tenant ready
	•	security focused
	•	mobile/tablet compatible
	•	API-driven frontend

⸻

Dashboard Goals

The dashboard is designed to support the following operational tasks:
	1.	Monitor incoming events
	2.	Monitor suspicious activity
	3.	View alerts and alert status
	4.	Manage investigation cases
	5.	Review evidence and logs
	6.	Monitor device status
	7.	View system statistics and trends
	8.	Manage users and organizations
	9.	Review audit logs
	10.	Configure detection rules and thresholds

⸻

Summary

	•	Real-time monitoring

	•	Investigation tools
	•	Evidence management
	•	Reporting and analytics

The dashboard should be designed as a modular, API-driven interface that can scale from MVP to enterprise-level deployment.
