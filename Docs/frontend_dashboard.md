NSD – Frontend Dashboard Design

Overview

This document describes the frontend dashboard design for NSD (Network Suspicious Detection).

The dashboard is used by operators, administrators, and analysts to monitor devices, alerts, suspicious activity, and system status in real time.

The main goal of the dashboard is:
	•	Visual monitoring
	•	Alert management
	•	Case investigation
	•	Device tracking
	•	System visibility

⸻

Dashboard Main Functions

The frontend dashboard should provide:
	1.	Real-time alerts
	2.	Device location map
	3.	Suspicious activity timeline
	4.	Alert and case management
	5.	Device management
	6.	User management (admin)
	7.	Reports and analytics
	8.	System status monitoring

Main Dashboard Screen

Dashboard Widgets

Main dashboard should show:
	•	Active Alerts Count
	•	High Risk Alerts
	•	Devices Online / Offline
	•	Suspicious Events Today
	•	Alerts Trend Graph
	•	System Status
	•	Recent Alerts List

Example widgets:

Widget	Description
Active Alerts	Number of active alerts
High Risk Alerts	Critical alerts
Devices Online	Connected devices
Devices Offline	Disconnected devices
Events Today	Suspicious events today
Alerts Trend	Alerts over time
System Status	API / DB / Engine status


⸻

Alerts Page

The Alerts page shows all alerts generated by the detection engine.

Alert Information

Each alert should contain:
	•	Alert ID
	•	Alert Type
	•	Risk Level
	•	Device ID
	•	Location
	•	Time
	•	Status (Open / Investigating / Closed)
	•	Assigned Operator

Alert Actions

Operators can:
	•	View alert details
	•	Assign alert
	•	Change status
	•	Create case
	•	Add notes
	•	Escalate alert

⸻

Cases Page

Cases are investigations created from alerts.

Case Information
	•	Case ID
	•	Related Alerts
	•	Devices
	•	Assigned Investigator
	•	Status
	•	Notes
	•	Timeline
	•	Evidence / Files

Case Status Examples:
	•	Open
	•	Investigating
	•	Monitoring
	•	Resolved
	•	Closed

⸻

Device Management Page

Device list with:
	•	Device ID
	•	Device Name
	•	Owner
	•	Status (Online / Offline)
	•	Last Seen
	•	Battery (if IoT)
	•	Location
	•	Risk Score

Actions:
	•	Register device
	•	Disable device
	•	View device history
	•	View movement history
	•	Assign to user
	•	Delete device

⸻

Map View

Map view is one of the most important features.

Map should display:
	•	Device locations
	•	Alert locations
	•	Geofences
	•	Movement history
	•	Suspicious activity locations

Map features:
	•	Real-time device movement
	•	Alert markers
	•	Filter by device
	•	Filter by risk level
	•	Time range filter
	•	Heatmap (optional future)

Possible map technologies:
	•	Google Maps
	•	Mapbox
	•	OpenStreetMap
	•	Leaflet.js

⸻

Reports & Analytics

Reports page should include:
	•	Alerts per day/week/month
	•	High risk alerts
	•	Device activity statistics
	•	Movement statistics
	•	Case resolution time
	•	Operator activity
	•	System performance

Charts:
	•	Line chart (alerts over time)
	•	Bar chart (alerts by type)
	•	Pie chart (risk levels)
	•	Heatmap (locations)
	•	Device activity graph

	•	Detection rules

	•	Notification settings
	•	System settings

	•	Device movement
	•	Status changes
	•	Notifications
	•	Dashboard widgets update

Summary

The NSD Frontend Dashboard is responsible for:
	•	Monitoring alerts
	•	Visualizing device locations
	•	Managing alerts and cases
	•	Viewing analytics and reports
	•	Managing devices and users
	•	Providing real-time system visibility

The dashboard acts as the main control center of the NSD system.
