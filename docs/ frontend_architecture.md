NSD – Frontend Architecture

Overview

The NSD Frontend is the user interface for monitoring, investigation, alert management, and system administration.

It provides operators, administrators, and analysts with dashboards, alerts, case management tools, device monitoring, reports, and system configuration interfaces.

The frontend communicates with the Backend API and WebSocket services to retrieve data and receive real-time alerts.

⸻

Frontend Architecture Overview

Browser / Mobile
        ↓
Frontend Application (React / Vue)
        ↓
API Gateway / Backend API
        ↓
Database / Detection Engine / Alert System

The frontend is a client-side application that interacts with backend services via REST APIs and real-time event streams.

API Communication

Frontend communicates with backend using REST APIs.

Example API Calls

Method	Endpoint	Purpose
GET	/api/dashboard	Dashboard data
GET	/api/alerts	Alert list
GET	/api/devices	Device list
GET	/api/cases	Case list
POST	/api/cases	Create case
PUT	/api/alerts/{id}	Update alert
GET	/api/reports	Reports
GET	/api/admin/users	User management
