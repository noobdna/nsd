NSD – Frontend Dashboard Design

Overview

The Frontend Dashboard is the main user interface for operators,
administrators, and analysts to monitor suspicious activities,
manage alerts, and investigate cases.

The dashboard should provide real-time visibility,
map-based tracking, alert management, and system status monitoring.

⸻

Main Objectives

The dashboard must allow users to:
	•	View devices and events on a map
	•	Monitor alerts in real time
	•	Investigate suspicious events
	•	Manage cases and assign operators
	•	View risk scores and analytics
	•	Manage devices and locations
	•	View system logs and audit trails

⸻

User Roles

Operator
	•	Monitor alerts
	•	View map and events
	•	Create and update cases
	•	Add notes

Administrator
	•	Manage users
	•	Manage devices
	•	Manage locations and geofences
	•	System configuration

Analyst
	•	View historical data
	•	Analyze movement patterns
	•	Export reports
	•	Risk analysis

⸻

Main Dashboard Screens

1. Main Dashboard

Displays system overview.

Widgets:
	•	Active alerts
	•	Devices online/offline
	•	Events in last 24 hours
	•	High risk alerts
	•	System status
	•	Map overview

⸻

2. Map View

Map-based monitoring interface.

Features:
	•	Device locations
	•	Event markers
	•	Alert markers
	•	Geofence areas
	•	Movement history
	•	Real-time updates
	•	Filter by device / alert level / time

⸻

3. Alerts Page

List of alerts generated by the system.

Alert Information:
	•	Alert ID
	•	Device
	•	Location
	•	Time
	•	Risk score
	•	Alert level (Low / Medium / High / Critical)
	•	Status (Open / Investigating / Closed)
	•	Assigned operator

Actions:
	•	View alert details
	•	Assign operator
	•	Change status
	•	Create case
	•	Add notes

⸻

4. Case Management Page

Used to manage investigations.

Case Information:
	•	Case ID
	•	Related alerts
	•	Assigned operator
	•	Status
	•	Notes
	•	Timeline of events
	•	Attachments / images / reports

⸻

5. Devices Management Page

Manage registered devices.

Device Info:
	•	Device ID
	•	Device type
	•	Owner
	•	Status
	•	Last seen
	•	Battery level
	•	Firmware version
	•	Location

⸻

6. Locations / Geofence Management

Manage monitored locations.

Location Info:
	•	Location name
	•	Latitude / Longitude
	•	Radius
	•	Type (School / Office / Restricted Area / Store / Home)
	•	Alert rules

⸻

7. Reports & Analytics Page

Used for analysis and reporting.

Reports:
	•	Alerts per day
	•	Alerts per location
	•	Device activity
	•	Movement heatmap
	•	Risk score distribution
	•	Case resolution time
	•	Export CSV / PDF

⸻

8. System Logs / Audit Page

Security and audit logs.

Logs include:
	•	User login
	•	Configuration changes
	•	Device registration
	•	Alert status changes
	•	Case updates
	•	System errors

Recommended Frontend Technologies

Web Dashboard
	•	React
	•	Next.js
	•	TypeScript
	•	Mapbox or Google Maps
	•	Chart.js / ECharts
	•	Tailwind CSS

Mobile Dashboard (Optional Future)
	•	React Native
	•	Flutter

⸻

Real-Time Features

Dashboard should support real-time updates using:
	•	WebSocket
	•	Server-Sent Events (SSE)
	•	Push notifications

Real-time updates include:
	•	New alerts
	•	Device movement
	•	Device offline
	•	Case updates
	•	System warnings

⸻

Security Requirements

Frontend must implement:
	•	Authentication (JWT / OAuth / SSO)
	•	Role-Based Access Control (RBAC)
	•	HTTPS only
	•	Session timeout
	•	Audit logging
	•	Sensitive data masking
	•	Device access logging
