NSD – Notification System

Overview

The Notification System is responsible for delivering alerts generated by the NSD detection engine to appropriate users, operators, and external systems.

The system ensures that suspicious events are not only detected but also communicated, tracked, acknowledged, and escalated if necessary.

The notification system supports multiple delivery channels and escalation workflows to ensure critical alerts are never missed.

⸻

Notification Flow

High-level flow:

Event Detected
      ↓
Detection Engine
      ↓
Alert Created
      ↓
Notification Engine
      ↓
Channel Selection
      ↓
Notification Delivery
      ↓
Acknowledgement / Escalation
      ↓
Audit Log


⸻

Notification Types

Notifications are generated from alerts and can be categorized by severity and urgency.

Severity Levels

Level	Description
Info	Informational event
Low	Minor suspicious activity
Medium	Suspicious behavior
High	Serious suspicious activity
Critical	Immediate action required

Severity determines:
	•	notification priority
	•	delivery channel
	•	escalation timing
	•	required acknowledgement

⸻

Notification Channels

The system supports multiple notification channels.

Supported Channels

Channel	Usage
Email	Standard alert notifications
SMS	Urgent alerts
Push Notification	Mobile app alerts
Dashboard	Alert center
Webhook	External integrations
Slack / Teams	Operations alerts
Voice Call	Critical alerts
PagerDuty	On-call escalation

The system can send notifications through multiple channels simultaneously depending on alert severity.

Example:

Severity	Channels
Info	Dashboard
Low	Dashboard + Email
Medium	Email + Push
High	Email + Push + SMS
Critical	SMS + Voice + PagerDuty

Notification Status Lifecycle

Notifications move through several states.

Status	Description
pending	Waiting to be sent
sent	Sent to channel
delivered	Successfully delivered
failed	Delivery failed
acknowledged	User acknowledged
escalated	Escalated to next level


⸻

Acknowledgement Workflow

Some alerts require acknowledgement from an operator.

Acknowledgement Flow

Notification Sent
        ↓
User Receives Notification
        ↓
User Acknowledges
        ↓
Alert Status Updated
        ↓
Case May Be Created

If not acknowledged within a defined time, escalation is triggered.

⸻

Escalation Policy

Escalation ensures that critical alerts are not ignored.

Example Escalation Policy

Severity	Escalation Time
Low	No escalation
Medium	30 minutes
High	10 minutes
Critical	2 minutes

Escalation Flow

Notification Sent
       ↓
Wait for Acknowledgement
       ↓
Timeout Reached
       ↓
Escalate to Next User / Team
       ↓
Send New Notification
       ↓
Log Escalation

Escalation targets may include:
	•	team lead
	•	security team
	•	administrator
	•	on-call engineer
	•	external service

Notification Routing

The system determines who receives notifications based on:
	•	alert severity
	•	organization
	•	device owner
	•	location
	•	risk score
	•	time of day
	•	on-call schedule
	•	escalation policy

Routing Example

Alert Type	Recipient
Device offline	Device owner
Suspicious login	Security team
Night access	Facility manager
Multiple failed logins	Security team

Retry and Failure Handling

If notification delivery fails:
	1.	Retry sending
	2.	Try alternative channel
	3.	Log failure
	4.	Escalate if necessary

Retry Strategy Example

Attempt	Action
1	Send Email
2	Retry Email
3	Send SMS
4	Escalate

Summary

The NSD Notification System ensures that alerts generated by the detection engine are delivered, acknowledged, escalated, and audited.

Core responsibilities:
	•	Deliver alerts to users
	•	Support multiple notification channels
	•	Manage acknowledgement workflow
	•	Handle escalation policies
	•	Retry failed deliveries
	•	Log all notification activity
	•	Ensure critical alerts are never missed

The notification system is a critical component connecting detection to human response and incident management.
