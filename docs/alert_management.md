NSD – Alert Management

Overview

The Alert Management module is responsible for handling alerts generated by the NSD detection engine.

Alerts represent suspicious events or behaviors that require attention, investigation, or automated response. The alert system allows operators to monitor suspicious activity, prioritize incidents, assign investigations, and track resolution status.

Alerts are generated from events after risk scoring and detection rule evaluation.

⸻

Alert Flow

The alert lifecycle in NSD follows this flow:

Event → Detection Engine → Risk Score → Alert → Case → Investigation → Resolution

	1.	Event is received from devices, logs, or sensors
	2.	Detection engine analyzes behavior patterns
	3.	Risk score is calculated
	4.	If threshold exceeded → Alert generated
	5.	Alert may be escalated to Case
	6.	Investigation performed
	7.	Case closed and alert resolved

⸻

Alert Severity Levels

Alerts are categorized by severity to help prioritize responses.

Severity	Description
low	Minor anomaly
medium	Suspicious behavior
high	Likely malicious activity
critical	Active incident / breach

Severity is determined by:
	•	risk score
	•	detection rule type
	•	asset importance
	•	repeated behavior
	•	time of activity
	•	location anomaly

⸻

Alert Status

Alerts move through different statuses during their lifecycle.

Status	Description
open	Alert created
acknowledged	Analyst acknowledged alert
investigating	Investigation in progress
escalated	Escalated to case
resolved	Issue resolved
closed	Alert closed
false_positive	Determined to be false alert

Alert Priority Logic

Alert priority can be calculated from multiple factors.

Example priority calculation factors:
	•	Risk Score
	•	Asset Importance
	•	Alert Frequency
	•	Time of Day
	•	Location Risk
	•	User Risk Score
	•	Device Risk Score
	•	Historical Incidents

Example Priority Formula

priority_score =
    risk_score
    + asset_value
    + user_risk
    + device_risk
    + location_risk
    + repeat_factor

Priority Levels:

Priority Score	Priority
0 – 30	low
31 – 60	medium
61 – 80	high
81+	critical

Assignment includes:
	•	security analyst
	•	investigator
	•	admin
	•	automated system
	•	escalation team

Assignment workflow:

Alert created
   ↓
Auto assignment OR manual assignment
   ↓
Investigation
   ↓
Escalate to case if needed


⸻

Alert Escalation to Case

An alert may be escalated into a case when:
	•	Severity is high or critical
	•	Multiple related alerts exist
	•	Investigation required
	•	Evidence collection required
	•	Incident confirmed
	•	Compliance reporting required

When escalated:
	•	Case is created
	•	Alert linked to case
	•	Evidence can be attached
	•	Investigation notes recorded
